## 画面名：ログイン

### 目的
社内利用のログイン画面

### 対象スキーマ
SalesUser

### 表示項目
- ID
- パスワード

### 機能
- credentials方式
- jwt方式
- パスワードはハッシュ化
- 認証判定
- バリデーション

### 挙動
- 他画面に認証していない状態でのログイン後のコールバック先
- 認証成功後は求職者一覧へ
- 認証失敗時についてもログイン画面

### ルーティング
ログイン /login
求職者一覧 /applicants

### エラーメッセージ
認証失敗 → 「IDまたはパスワードが違います」

バリデーション → 各項目の下に表示

### ルール

| 対象（画面） | エンティティ | 項目名 | 必須か | 入力制約（文字数・形式など） | 権限方針（誰が操作可能か / MVP） | 履歴方針（残す/残さない） | 一覧検索対象か | 一覧の並び順キーか | 備考 |
|--------------|--------------|--------|--------|------------------------------|----------------------------------|----------------------------|------------------|----------------------|------|
| ログイン | 営業担当者 | ID | Yes | 1〜50文字、英数字と記号（-_）のみ | 編集不可 | - | - | - | - |
| ログイン | 営業担当者 | パスワード | Yes | 8文字以上 | 編集不可 | - | - | - | - |

### スキーマ

```
model SalesUser {
  // ✅ ログインID（1〜50文字、英数字と - _ のみ）を“手入力で”使う想定
  // cuid() や autoincrement() は使わない
  id       String   @id @db.VarChar(50)
  name     String   @db.VarChar(255)
  password String   @db.VarChar(255)
  isActive Boolean  @default(true)

  jobSeekers JobSeeker[]

  @@map("sales_users")
}
```
